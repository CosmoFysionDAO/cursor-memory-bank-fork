## Безопасность агента (Agent Security)

- **.cursorignore**: Файлы и пути, перечисленные в `.cursorignore`, не должны передаваться в контекст агента и не должны изменяться агентом. Проверяйте наличие паттернов для `.env*`, credentials, ключей (`*.key`, `*.pem`), каталогов `**/private/**`, `**/secrets/**`.
- **Секреты в коде**: Не вставлять реальные ключи, пароли или токены в код. Использовать переменные окружения или секрет-менеджеры; в примерах — плейсхолдеры (`<API_KEY>`, `$ENV_VAR`).
- **Правила Cursor**: Соблюдать ограничения и инструкции из `.cursor/rules/` (в том числе security-правила). Учитывать приоритет: Team Rules > Project Rules (`.cursor/rules`) > User Rules > AGENT.md.
- **Обновление**: При изменении структуры проекта или появлении новых чувствительных путей обновлять `.cursorignore` и при необходимости раздел безопасности в AGENT.md. Для проверки использовать команду `/security` (Check Security Agent Mode).
